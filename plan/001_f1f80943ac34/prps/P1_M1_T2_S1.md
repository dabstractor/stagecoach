# PRP — P1.M1.T2.S1: internal/ui/exitcode.go — canonical exit codes

## Goal

**Feature Goal**: Define the single, canonical set of process exit-code constants for stagehand in `internal/ui/exitcode.go`, frozen to the exact integer values mandated by PRD §15.4, so every error/timeout/rescue/nothing-to-commit path across the binary maps to the same codes.

**Deliverable**: A new file `internal/ui/exitcode.go` (package `ui`) exporting five typed `int` constants (`ExitSuccess`, `ExitError`, `ExitNothingToCommit`, `ExitRescue`, `ExitTimeout`) with explicit integer literals, plus `internal/ui/exitcode_test.go` — a golden, table-driven test asserting each constant equals its exact PRD §15.4 value. This is the first file under `internal/` and the first `_test.go` in the repo.

**Success Definition**: `go test ./internal/ui/` passes and asserts ExitSuccess==0, ExitError==1, ExitNothingToCommit==2, ExitRescue==3, ExitTimeout==124 exactly; `go build ./internal/ui/` and `go vet ./internal/ui/` are clean; the file imports nothing (pure constants). `cmd/stagehand` (and any future package) can import `github.com/dustin/stagehand/internal/ui` and pass these consts to `os.Exit`.

## Why

- **PRD §15.4 is a binding fixed code table.** Users, shell scripts, and CI read stagehand's exit code to branch behavior (e.g. `stagehand && ...`, `timeout 120 stagehand`). A silent shift (2↔3, or Timeout becoming 4) breaks every consumer.
- **Cross-task contract freeze.** decisions.md §3 (the two-nested-loop orchestrator) routes every failure to a RESCUE branch; decisions.md §7 lists the CAS-failure path as "exit 1"; M7.T2.S1 maps cobra RunE errors to these codes; M6.T2.S1 (rescue) and the timeout path return ExitRescue / ExitTimeout. All of that depends on these constants being stable — hence the golden test.
- **Establishes the `internal/ui` package** (sibling `output.go` arrives in P1.M1.T2.S2) and the repo-wide testing pattern (stdlib `testing`, table-driven).
- **124 is deliberate**, not arbitrary: it mirrors the GNU `timeout` utility's own exit-124-on-timeout convention (PRD §15.4), so a stagehand invocation wrapped under `timeout` reports timeout consistently.

## What (technical requirements)

1. New file `internal/ui/exitcode.go`, `package ui`.
2. Five exported, typed `int` constants with EXPLICIT integer literals (NOT iota):
   - `ExitSuccess = 0`
   - `ExitError = 1`
   - `ExitNothingToCommit = 2`
   - `ExitRescue = 3`
   - `ExitTimeout = 124`
3. Each constant carries a doc comment naming the PRD §15.4 meaning and at least one triggering condition.
4. No imports, no functions, no logic — a pure constants file.
5. New file `internal/ui/exitcode_test.go` (`package ui`, white-box) with a table-driven golden test asserting the exact values above.

```go
package ui
import "testing"
func TestExitCodes(t *testing.T) {
	tests := []struct {
		name string
		got  int
		want int
	}{
		{"ExitSuccess", int(ExitSuccess), 0},
		{"ExitError", int(ExitError), 1},
		{"ExitNothingToCommit", int(ExitNothingToCommit), 2},
		{"ExitRescue", int(ExitRescue), 3},
		{"ExitTimeout", int(ExitTimeout), 124},
	}
	for _, tc := range tests {
		tc := tc
		t.Run(tc.name, func(t *testing.T) {
			if tc.got != tc.want {
				t.Errorf("%s = %d, want %d", tc.name, tc.got, tc.want)
			}
		})
	}
}

```
