# PRP — P1.M3.T1.S2: internal/git/gittestutil_test.go — temp-repo harness

## Goal

**Feature Goal**: Build the shared test infrastructure for `package git`: a temp-repo harness that, against the REAL host git binary (git 2.54.0), spins up an isolated repository per test, configures a deterministic commit identity, stages known content, and seeds a deterministic linear commit history with arbitrary full (possibly multi-line) messages. Every downstream `internal/git` test (plumbing T2, diff T3, log/stage T4) and the generate integration suite (M6.T3) will compose this harness instead of reinventing `git init`/`git add`/`git commit` boilerplate. Standing it up as its own unit (plan_overview §8) keeps that boilerplate in ONE place so a git-version or identity quirk is fixed once, not in every test.

**Deliverable**: ONE NEW file `internal/git/gittestutil_test.go` (white-box `package git`, stdlib `testing` ONLY, no build tag, no external test deps). It exposes three unexported test helpers — `newTempRepo`, `seedCommits`, `writeFileStage` — plus a self-test `TestHarness_BootstrapsRepoWithCommits` that proves the harness actually works. It consumes the shipped S1 API (`git.New`, `(g *Git) run`, `g.dir`) and changes NOTHING else.

**Success Definition**: `go vet ./internal/git/` clean (this is the gate that compiles `_test.go`), `gofmt -l internal/git/` empty, `go test ./internal/git/` PASS (self-test bootstraps a repo with ≥2 commits and `rev-parse HEAD` succeeds; the shipped S1 tests still pass), `go test ./...` whole-module green. The helpers drive the REAL git binary over a per-test temp dir auto-removed by `tb.TempDir()`. No external dep added; main.go/Makefile/go.mod/go.sum/internal/ui/internal/provider and the shipped git.go/git_test.go untouched; no `go mod tidy`.

## Why

- **Test infra stands as its own unit (plan_overview §8; decisions.md §8).** Every `internal/git/*_test.go` (T2 plumbing, T3 diff, T4 log/stage) and the generate integration suite (M6.T3) needs a temp repo + staged content + a deterministic commit history. Putting that setup in ONE harness file means it is written/debugged ONCE, and a single fix (e.g. a git-2.x default, a gpg quirk) propagates everywhere instead of being scattered across N test files.
- **The harness drives the REAL git binary (PRD §20.1 layer 2).** That is the entire point of layer 2: "Unit — git wrapper, with a real git binary. Each `internal/git/*` test creates a temp directory, `git init`, stages known content, and asserts on `WriteTree`/`CommitTree`/...". The harness does NOT mock git — it mocks only the REPO (temporary + deterministic identity), so it catches real plumbing regressions in the binary our production code shells out to (PRD §22.3).
- **White-box by necessity.** The shipped `run` is UNEXPORTED and `g.dir` is UNEXPORTED (S1 contract, confirmed in internal/git/git.go). The harness calls `g.run(...)` and reads `g.dir`, so it MUST be `package git` (white-box), exactly like the shipped `internal/git/git_test.go`. Because `_test.go` symbols are not importable across packages, this also forces every downstream `internal/git` test to be `package git` white-box to reach the helpers — which is the intended posture.
- **Deterministic identity removes a whole flake class.** `git commit` fails without `user.email`/`user.name`; a host with `--global commit.gpgsign true` would make every harness commit try to sign and flake in CI. Setting repo-local deterministic identity (+ a defensive repo-local `commit.gpgsign false`) makes every test reproducible on every developer machine, which is the harness's whole purpose. Repo-local config never touches production repos (production inherits the user's real config).

2. **`newTempRepo(tb testing.TB) *Git`** — the repo factory.
   - `tb.Helper()` first (mark as a test helper so a failure reports the CALLER).
   - `dir := tb.TempDir()` (stdlib testing: returns a unique dir the testing package auto-removes via an internal cleanup at test end — this SATISFIES the contract's "t.Cleanup removes dir"; do NOT call `os.MkdirTemp` + manual `t.Cleanup(os.RemoveAll)` — that reinvents `tb.TempDir`).
   - `g, err := New(dir)`; on error `tb.Fatalf("git.New: %v", err)`.
   - Bootstrap the repo + identity by driving the REAL binary through the shipped seam: `mustRun(tb, g, "init", "-q")`; `mustRun(tb, g, "config", "user.email", "stagehand@example.com")`; `mustRun(tb, g, "config", "user.name", "Stagehand Test")`; `mustRun(tb, g, "config", "commit.gpgsign", "false")` (defensive cross-machine determinism — see Known Gotchas).
   - Return `g` over an UNBORN repo (NO initial commit) so T2 root-commit tests observe `RevParseHEAD` returning ok=false. Tests needing history call `seedCommits` themselves.
   - godoc: cites PRD §20.1 layer 2 (real git binary + temp repo) and decisions.md §8 (shared harness).
3. **`writeFileStage(tb testing.TB, g *Git, path string, content string)`** — write a file under the repo and stage it.
   - `tb.Helper()`.
   - `path` is RELATIVE to the repo root (e.g. `"f.go"`, `"pkg/sub/f.go"`). Compute `full := filepath.Join(g.dir, path)` (white-box read of `g.dir`).
   - `if err := os.MkdirAll(filepath.Dir(full), 0o755); err != nil { tb.Fatalf(...) }` (so nested subdirectory paths work — T3 diff tests nest files).
   - `if err := os.WriteFile(full, []byte(content), 0o644); err != nil { tb.Fatalf(...) }`.
   - `mustRun(tb, g, "add", path)` — stage the RELATIVE path (cmd.Dir is g.dir, so git resolves it inside the repo; the staged path then matches the on-disk path).
   - godoc: writes the file then `git add` (real binary, PRD §20.1 layer 2).
4. **`seedCommits(tb testing.TB, g *Git, msgs []string)`** — create a deterministic linear history of N commits, one full message each.
   - `tb.Helper()`.
   - For `i, msg := range msgs`: `writeFileStage(tb, g, fmt.Sprintf("file%d.txt", i), fmt.Sprintf("content %d\n", i))` (a UNIQUE file per commit so every commit is non-empty and the tree actually changes → successive commits are distinct and linear), then `mustRun(tb, g, "commit", "-q", "-m", msg)`.
   - A SINGLE `-m` arg carries the FULL message, including embedded newlines/paragraphs (verified: `git log --format=%B` reproduces multi-line bodies exactly). Do NOT use multiple `-m` flags (those become separate paragraphs) and do NOT use `-F`/stdin (`g.run` has `cmd.Stdin=nil`). This is what T4 `RecentMessages` multi-line tests depend on.
   - godoc: creates one commit per message via the real binary; pairs with writeFileStage so each commit has distinct staged content.
5. **`mustRun(tb testing.TB, g *Git, args ...string)`** — a tiny private fail-fast wrapper: `tb.Helper(); out, err := g.run(args...); if err != nil { tb.Fatalf("git %v: %v\nstdout:%s", args, err, out) }`. Keeps the three helpers readable and guarantees a git error aborts the test with a clear message (no silent corruption). (Alternatively inline the check in each helper — either is acceptable; `mustRun` avoids repetition.)
6. **Self-test `TestHarness_BootstrapsRepoWithCommits(t *testing.T)`** — proves the harness works end-to-end against the real binary:
   - `g := newTempRepo(t)`.
   - `seedCommits(t, g, []string{"feat: first commit\n\nFirst body line.\nSecond body line.", "fix: second commit"})` (one multi-line + one single-line message; ≥2 as the contract requires).
   - `out, err := g.run("rev-list", "--count", "HEAD")`; `err==nil`; `count, e := strconv.Atoi(strings.TrimSpace(out))`; `e==nil` and `count >= 2`.
   - `head, err := g.run("rev-parse", "HEAD")`; `err==nil`; `len(strings.TrimSpace(head)) == 40` (full SHA).
   - Deterministic identity readback: `email, _ := g.run("config", "user.email")`; `strings.TrimSpace(email) == "stagehand@example.com"`; similarly `user.name == "Stagehand Test"`.
   - One behavior per `Test*`; this is the harness's own regression guard.
7. **Scope / boundary**: ONLY create `internal/git/gittestutil_test.go`. Do NOT modify the shipped `internal/git/git.go` or `internal/git/git_test.go`, main.go, Makefile, go.mod, go.sum, internal/ui, internal/provider. Do NOT run `go mod tidy`. ### Implementation Patterns & Key Details
```go
func mustRun(tb testing.TB, g *Git, args ...string) {
	tb.Helper()
	out, err := g.run(args...)
	if err != nil {
		tb.Fatalf("git %v: %v\nstdout:%s", args, err, out)
	}
}
func newTempRepo(tb testing.TB) *Git {
	tb.Helper()
	dir := tb.TempDir()
	g, err := New(dir)
	if err != nil {
		tb.Fatalf("git.New: %v", err)
	}
	mustRun(tb, g, "init", "-q")
	mustRun(tb, g, "config", "user.email", "stagehand@example.com")
	mustRun(tb, g, "config", "user.name", "Stagehand Test")
	mustRun(tb, g, "config", "commit.gpgsign", "false") 
	return g 
}
func writeFileStage(tb testing.TB, g *Git, path, content string) {
	tb.Helper()
	full := filepath.Join(g.dir, path) 
	if err := os.MkdirAll(filepath.Dir(full), 0o755); err != nil {
		tb.Fatalf("mkdir %s: %v", filepath.Dir(full), err)
	}
	if err := os.WriteFile(full, []byte(content), 0o644); err != nil {
		tb.Fatalf("write %s: %v", full, err)
	}
	mustRun(tb, g, "add", path) 
}
func seedCommits(tb testing.TB, g *Git, msgs []string) {
	tb.Helper()
	for i, msg := range msgs {
		writeFileStage(tb, g, fmt.Sprintf("file%d.txt", i), fmt.Sprintf("content %d\n", i))
		mustRun(tb, g, "commit", "-q", "-m", msg)
	}
}

```
