# PRP for P1.M7.T1.S1

## Objective

Ship the tiny, stable public library surface in pkg/stagehand (PRD §14.1): the exported Options + Result types and the GenerateCommit(ctx, opts) (Result, error) function — a thin wrapper that builds resolved Config+Registry internally, resolves provider+model (with first-detected fallback), delegates to internal/generate.CommitStaged, and maps the internal Result to the public Result (adding resolved Provider/Model). Includes the additive DryRun (PRD FR49) and SystemExtra seams on internal/generate.Deps/CommitStaged that the wrapper requires, plus godoc `// Stable as of v1.0` markers (Appendix E.6). NEVER stages (v2 seam); NEVER os.Exit (returns sentinel errors).

## Context

## Goal

**Feature Goal**: A tested, self-contained `pkg/stagehand/stagehand.go` exposing the v1-stable public API (PRD §14.1): `Options`, `Result`, and `GenerateCommit(ctx, opts) (Result, error)`. GenerateCommit is a thin wrapper that resolves provider+model, delegates to `internal/generate.CommitStaged` (the shipped two-nested-loop orchestrator), supports `DryRun` (run the full pipeline but skip commit-tree/update-ref; return the message with CommitSHA="", PRD FR49) and `SystemExtra` (append to the built system prompt), and NEVER stages (v2 seam, decisions.md §1).

**Deliverable**:
1. `pkg/stagehand/stagehand.go` — owns `// Package stagehand`; exports `Options`, `Result`, `GenerateCommit`, and re-exports the three sentinel errors as aliases; each exported symbol carries godoc `// Stable as of v1.0`.
2. `pkg/stagehand/stagehand_test.go` — integration tests (real git temp repo + compiled stub agent): DryRun (CommitSHA=="", HEAD unchanged, message returned) and default (commit created, Provider/Model resolved).
3. ADDITIVE seams in `internal/generate/generate.go`: `Deps.DryRun bool` + `Deps.SystemExtra string` and two short-circuits in CommitStaged (behavior-identical when zero-valued).
4. ADDITIVE unit tests in `internal/generate/generate_test.go`: `TestCommitStaged_DryRun` + `TestCommitStaged_SystemExtra` (hermetic stubGit/stubRunner).

**Success Definition**: `go vet ./pkg/stagehand/ ./internal/generate/`, gofmt clean, and `go test ./...` all pass; DryRun returns the message + empty CommitSHA + no commit (HEAD unchanged); the default path creates a real commit (Result.CommitSHA == new HEAD); Result.Provider/Model are the resolved values; existing M6.T1.S1 CommitStaged tests stay green (seams are additive).

## Why
- This is the public integration seam for US12 (embed commit generation in a GUI/CI/pre-commit tool) and the single delegate target of the CLI default action (P1.M7.T2.S1: parse flags → maybe auto-stage → stagehand.GenerateCommit → exit codes).
- Keeps the CLI a thin shell over the library (PRD §14.1) and guarantees v2 can reuse GenerateCommit in a loop (multi-commit decomposition is v2; the seam — CommitStaged assumes the index is already staged — is preserved here because GenerateCommit NEVER stages).
- Closes the two gaps the shipped CommitStaged has relative to the public Options contract: DryRun (FR49) and SystemExtra (§14.1).

## What

### Public API (PRD §14.1, verbatim field set — additive-only)
```go

type Options struct {
    Provider    string        
    Model       string        
    SystemExtra string        
    DryRun      bool          
    Timeout     time.Duration 
}
type Result struct {
    CommitSHA string 
    Subject   string
    Message   string 
    Provider  string 
    Model     string 
}
func GenerateCommit(ctx context.Context, opts Options) (Result, error)

```

### Behavior
- GenerateCommit builds resolved Config+Registry internally via `config.Load(config.Flags{}, ".")` (repoDir = cwd), then layers `opts` on top (opts is HIGHEST precedence, above config files). It deliberately does NOT read STAGEHAND_* env (env/flag parsing is the CLI layer's job, T2.S1 folds env→opts).
- Resolves provider: opts.Provider → cfg.Provider → first detected provider in `reg.List()` order (`reg.Detect()`). If none detected/configured → error. Resolves model: opts.Model → cfg.Model → manifest.DefaultModel.
- Wires `git.New(".")`, `provider.NewExecutor(".")`, `ui.NewOutput(os.Stdout, os.Stderr, false, false)` (verbose=false; color auto-disables under a pipe so `--dry-run | tee` stays byte-clean).
- Builds `generate.Deps{..., DryRun: opts.DryRun, SystemExtra: opts.SystemExtra}` and calls `generate.CommitStaged(ctx, deps)`. On error, returns `Result{}` + the sentinel (propagated as-is; the CLI maps exit codes via errors.Is). On success, maps to public `Result{..., Provider: resolvedProvider, Model: resolvedModel}`.
- DryRun (FR49): CommitStaged runs diff→snapshot→generate→parse→dedupe, then short-circuits BEFORE commit-tree/update-ref, prints the message to stdout, returns `Result{CommitSHA:"", Subject, Message}`. It NEVER stages.

### DOCS impact (Mode A — per-item godoc)
- Each exported symbol (`Options`, `Result`, `GenerateCommit`, and the re-exported `Err*`) carries godoc `// Stable as of v1.0` (PRD Appendix E.6). Options stays additive-only. No separate docs/ file is owned by this item (CLI flag reference is owned by T2.S1 / M7.T3).

### Success Criteria
- [ ] DryRun: Result.CommitSHA=="", Subject+Message set, HEAD SHA UNCHANGED (no commit/ref mutation), Provider+Model resolved, err==nil.
- [ ] Default: Result.CommitSHA == new `git rev-parse HEAD` (40 hex), the commit appears in `git log` with the generated subject, Provider+Model resolved.
- [ ] `// Stable as of v1.0` present on Options, Result, GenerateCommit.
- [ ] Existing `internal/generate` tests stay green (seams are additive; DryRun=false/SystemExtra="" ⇒ identical M6.T1.S1 behavior).
- [ ] GenerateCommit NEVER calls git add/AddAll/HasStagedChanges and NEVER os.Exit.

## Implementation Blueprint

### Data models and structure
```go
package stagehand
import (
    "context"
    "fmt"
    "os"
    "time"
    "github.com/dustin/stagehand/internal/config"
    "github.com/dustin/stagehand/internal/generate"
    "github.com/dustin/stagehand/internal/git"
    "github.com/dustin/stagehand/internal/provider"
    "github.com/dustin/stagehand/internal/ui"
)
var (
    ErrNothingToCommit = generate.ErrNothingToCommit
    ErrRescue          = generate.ErrRescue
    ErrHeadMoved       = generate.ErrHeadMoved
)
func GenerateCommit(ctx context.Context, opts Options) (Result, error) {
    cfg, reg, _, err := config.Load(config.Flags{}, ".")
    if err != nil { return Result{}, err }
    if opts.Provider != "" { cfg.Provider = opts.Provider }
    if opts.Model != ""    { cfg.Model = opts.Model }
    if opts.Timeout > 0     { cfg.Timeout = opts.Timeout }
    resolvedProvider := cfg.Provider
    if resolvedProvider == "" {
        detected := reg.Detect()
        for _, n := range reg.List() {
            if detected[n] { resolvedProvider = n; break }
        }
    }
    if resolvedProvider == "" {
        return Result{}, fmt.Errorf("stagehand: no provider configured and no built-in agent detected on PATH")
    }
    manifest, ok := reg.Get(resolvedProvider)
    if !ok {
        return Result{}, fmt.Errorf("stagehand: provider %q not found", resolvedProvider)
    }
    resolvedModel := cfg.Model
    if resolvedModel == "" { resolvedModel = manifest.DefaultModel }
    cfg.Provider, cfg.Model = resolvedProvider, resolvedModel
    g, err := git.New(".")
    if err != nil { return Result{}, err }
    deps := generate.Deps{
        Git: g, Runner: provider.NewExecutor("."), Manifest: manifest, Config: cfg,
        Output: ui.NewOutput(os.Stdout, os.Stderr, false, false),
        DryRun: opts.DryRun, SystemExtra: opts.SystemExtra,
    }
    res, err := generate.CommitStaged(ctx, deps)
    if err != nil { return Result{}, err }
    return Result{CommitSHA: res.CommitSHA, Subject: res.Subject, Message: res.Message,
        Provider: resolvedProvider, Model: resolvedModel}, nil
}

```
