# PRP for P1_M1_T2_S2

## Objective

Create internal/ui/output.go defining a TTY/color-aware Output writer (package ui) that keeps stdout pipe-clean for `stagehand --dry-run | tee` (FR51) by routing ALL progress/verbose output to stderr and only result output to stdout. Deliver an `Output` struct (stdout/stderr io.Writers + resolved `color` bool + `verbose` bool) with methods Progressf‚Üístderr (always, FR18 auto-stage notice), Resultf‚Üístdout (FR42 `[sha] subject` + diff-tree lines), Verbosef‚Üístderr-only-when-verbose (FR50), plus a dependency-free minimal ANSI helper and a pure `shouldColor` predicate. Color is enabled iff stdout is a TTY AND `NO_COLOR` is unset/empty AND the `noColor` flag is false (FR51); implemented with raw `\x1b[..m` sequences and stdlib `os.ModeCharDevice` TTY detection ‚Äî NO external color/term library. White-box table-driven tests (internal/ui/output_test.go) inject `bytes.Buffer` writers + use `t.Setenv` to toggle `NO_COLOR`, asserting Progressf lands on stderr, Resultf on stdout, Verbosef is suppressed when verbose==false, and ANSI codes are absent unless color is forced on. DOCS impact: none (README is Mode B, M8.T4).

## Goal

**Feature Goal**: Provide stagehand's single user-facing output abstraction ‚Äî a TTY/color-aware `Output` writer in `internal/ui/output.go` that enforces the FR51 stream discipline: progress and verbose diagnostics go to **stderr always**, only the commit result (subject + diff-tree) goes to **stdout**, so stdout stays byte-clean for piping (`stagehand --dry-run | tee /tmp/msg.txt`). Color is auto-enabled on a TTY and disabled by `NO_COLOR` / `--no-color`, with a zero-dependency ANSI helper.

**Deliverable**: A new file `internal/ui/output.go` (package `ui`, NO package-doc comment ‚Äî that lives in the sibling exitcode.go) exporting: an `Output` struct; a constructor `NewOutput(stdout, stderr io.Writer, verbose, noColor bool) *Output`; the three routing methods `Progressf`, `Resultf`, `Verbosef`; the minimal ANSI helpers (`Color` + thin color wrappers); a pure `shouldColor(isTTY, noColorEnv, noColorFlag) bool` predicate; and a private `isTerminal(io.Writer) bool` using `os.ModeCharDevice`. Plus `internal/ui/output_test.go` (package `ui`, white-box) ‚Äî table-driven tests covering the pure color predicate, stream routing, verbose-suppression, and ANSI-absence/presence.

**Success Definition**: `go test ./internal/ui/` passes; tests prove Progressf‚Üístderr, Resultf‚Üístdout, Verbosef suppressed when verbose==false, `shouldColor` true only when (TTY && NO_COLOR empty && !noColor), and `Color()` emits ANSI iff `color` is true. `go build ./internal/ui/` and `go vet ./internal/ui/` clean; `gofmt -l internal/ui/` empty. Imports are stdlib-only (fmt, io, os) ‚Äî no cobra/pflag/color/term, no go.mod/go.sum change. The package-doc-comment trap is avoided (plain `package ui` in output.go).

## Why

- **FR51 stdout-clean invariant is product-critical.** The PRD's own example `stagehand --dry-run --no-color | tee /tmp/msg.txt` (¬ß15.5) only works if NOTHING but the generated message ever touches stdout. If a progress line ("staging 3 files‚Ä¶") leaks to stdout, the piped file is corrupted. Centralizing all writes behind `Output` makes that invariant structural rather than convention-dependent.
- **FR50/FR18 routing rules.** FR18 (auto-stage notice), FR50 (verbose: resolved command + raw agent stdout + retries), and FR42 (success result) each name a specific stream. `Output` encodes those three destinations as three methods so callers cannot misroute.
- **decisions.md ¬ß3 dependency.** The generate orchestrator's RESCUE branch and the CLI's auto-stage notice both print through a UI writer; this `Output` is that writer. It is consumed by cmd/stagehand (M7.T2) and generate/rescue (M6.T1.S3, M6.T2.S1).
- **Foundation + zero new deps.** Completes the `internal/ui` package (sibling exitcode.go already done). Color via raw ANSI + stdlib TTY stat keeps the dependency surface at exactly cobra + go-toml/v2 (external_deps.md ¬ßA); no `fatih/color`, `mattn/go-isatty`, or `golang.org/x/term`.

### Implementation Patterns
```go
package ui
import (
    "fmt"
    "io"
    "os"
)
const (
    ansiReset  = "\x1b[0m"
    ansiBold   = "\x1b[1m"
    ansiRed    = "\x1b[31m"
    ansiGreen  = "\x1b[32m"
    ansiYellow = "\x1b[33m"
    ansiCyan   = "\x1b[36m"
)
type Output struct {
    stdout  io.Writer
    stderr  io.Writer
    color   bool
    verbose bool
}
func NewOutput(stdout, stderr io.Writer, verbose, noColor bool) *Output {
    return &Output{
        stdout:  stdout,
        stderr:  stderr,
        color:   shouldColor(isTerminal(stdout), os.Getenv("NO_COLOR"), noColor),
        verbose: verbose,
    }
}
func shouldColor(isTTY bool, noColorEnv string, noColorFlag bool) bool {
    return isTTY && noColorEnv == "" && !noColorFlag
}
func isTerminal(w io.Writer) bool {
    f, ok := w.(*os.File)
    if !ok {
        return false
    }
    fi, err := f.Stat()
    if err != nil {
        return false
    }
    return fi.Mode()&os.ModeCharDevice != 0
}
func (o *Output) Progressf(format string, args ...any) error {
    _, err := fmt.Fprintf(o.stderr, format, args...)
    return err
}
func (o *Output) Resultf(format string, args ...any) error {
    _, err := fmt.Fprintf(o.stdout, format, args...)
    return err
}
func (o *Output) Verbosef(format string, args ...any) error {
    if !o.verbose {
        return nil
    }
    _, err := fmt.Fprintf(o.stderr, format, args...)
    return err
}
func (o *Output) Color(ansi, s string) string {
    if !o.color {
        return s
    }
    return ansi + s + ansiReset
}
func (o *Output) Green(s string) string  { return o.Color(ansiGreen, s) }
func (o *Output) Yellow(s string) string { return o.Color(ansiYellow, s) }
func (o *Output) Red(s string) string    { return o.Color(ansiRed, s) }
func (o *Output) Bold(s string) string   { return o.Color(ansiBold, s) }
func (o *Output) Cyan(s string) string   { return o.Color(ansiCyan, s) }

```
