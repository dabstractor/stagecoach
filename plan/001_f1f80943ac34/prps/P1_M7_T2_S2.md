## Context

# PRP — P1.M7.T2.S2: maybeAutoStage — auto-stage-all & nothing-staged CLI path

## Goal

**Feature Goal**: The single staging entry-point `maybeAutoStage(g, out, cfg, allFlag, noAutoStage) error` implements FR16-FR20 in the CLI layer: `--all`/`-a` forces `git add -A` even when something is already staged (FR20); if nothing is staged and auto-staging is enabled it runs `git add -A`, prints `Nothing staged — staging all changes (N files).` (FR18, WITH the file count), re-checks, and returns `ErrNothingToCommit` if STILL clean (FR17) or proceeds; if nothing is staged and the user declined auto-staging (`--no-auto-stage` OR `cfg.AutoStageAll=false`) it returns the DISTINCT sentinel `ErrNothingStaged` (FR19) without touching the index. It runs BEFORE `GenerateCommit` so the snapshot captures the staged set.

**Deliverable**: (a) ADD `(*Git).StagedFileCount() (int, error)` to `internal/git/stage.go` (+ tests in `internal/git/stage_test.go`) — the minimal file-count primitive; (b) CREATE `cmd/stagehand/stage.go` (plain `package main`) owning the `ErrNothingStaged` sentinel, the `stager` interface (extended with `StagedFileCount`), and the reshaped `func maybeAutoStage(...) error`; (c) EDIT `cmd/stagehand/run.go` — DELETE the old `maybeAutoStagePolicy` (+ its `stager` interface), change `runDefault`'s call site to `maybeAutoStage` + `mapErrorToExitCode`, and add the `ErrNothingStaged` branch to `mapErrorToExitCode`; (d) CREATE `cmd/stagehand/stage_test.go` — white-box `package main` MOCKING tests (7 cases incl. the missing happy path). DO NOT touch `pkg/stagehand`, `internal/generate`, or any provider/config code.

**Success Definition**: `gofmt -l`, `go build ./...`, `go vet`, and `go test ./...` all pass; the staging MOCKING contract is satisfied (nothing-staged+auto→AddAll+`(N files)` notice+proceed nil; clean-after-add→ErrNothingToCommit; --no-auto-stage+nothing→ErrNothingStaged+no add; --all force-adds on top of staged); both sentinels map to exit 2; `GenerateCommit` is never called by the staging path and never stages.

## Why

- This is the **sole staging entry-point** the v1 default action calls; v2 swaps it whole for a partitioned-staging loop (decisions.md §1, plan_overview key decision 6). A single, named, contract-faithful function is what makes that swap clean.
- P1.M7.T2.S1 (Complete) inlined a close-but-divergent `maybeAutoStagePolicy` because S2 was not yet done. S1's research note EXPLICITLY flagged maybeAutoStage as "sibling S2 test surface" and annotated the FR18 line `// FR18 (file count optional)` — i.e. S2 is the planned reconciliation, NOT a rework of broken code.
- The DISTINCT `ErrNothingStaged` (FR19) vs `ErrNothingToCommit` (FR17) lets integrators/tests branch via `errors.Is` even though both map to exit 2; the file count makes the FR18 notice actionable.

## What

### User-visible behavior (unchanged exits, refined messaging)
- `stagehand` with nothing staged + `auto_stage_all=true` (default): runs `git add -A`, prints `Nothing staged — staging all changes (N files).` to **stderr** (FR18), re-checks; still clean → `Nothing to commit.` + exit 2 (FR17); otherwise proceeds to generation.
- `stagehand --no-auto-stage` with nothing staged: prints `Nothing staged; nothing to commit.` to stderr, exits 2 (FR19), NEVER runs `git add`.
- `stagehand --all`/`-a`: forces `git add -A` even when something is already staged (FR20).
- Already staged, no `--all`: proceeds silently (exit 0 path) without calling `AddAll`.

### Success Criteria
- [ ] `maybeAutoStage` is THE sole staging entry-point; nothing else in `cmd/stagehand` calls `git add`/`AddAll` for the default action.
- [ ] `maybeAutoStage` returns `error` (nil=proceed): `ErrNothingStaged` (FR19), `ErrNothingToCommit` (FR17), or a wrapped git failure (→exit 1).
- [ ] The FR18 notice contains the file count, e.g. `Nothing staged — staging all changes (3 files).`
- [ ] `--no-auto-stage`+nothing and `AutoStageAll=false`+nothing both return `ErrNothingStaged` WITHOUT calling `AddAll`.
- [ ] `mapErrorToExitCode(ErrNothingStaged) == ui.ExitNothingToCommit (2)`; the PRD §15.4 exit-code table is unchanged.
- [ ] `GenerateCommit`/`CommitStaged` are never reached by the staging-error paths and never stage (v2 seam intact).
