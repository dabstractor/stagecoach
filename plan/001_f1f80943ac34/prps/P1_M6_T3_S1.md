# PRP for P1.M6.T3.S1

## Objective

Create the fake-agent stub provider test asset for the generate integration suite: internal/generate/testdata/stubagent/main.go (a standalone package-main binary that reads stdin, reads a JSON response 'script' + a cross-process counter file + an optional stdin-capture path from env, and per-invocation emits a canned valid message / empty-or-garbage / hang / non-zero-exit, configurable per-test) plus internal/generate/stubprovider_test.go (package generate) exposing BuildStubBinary, NewStubManifest, StubConfig, StubResponse and a TestStubProvider_* self-test suite asserting every behavior THROUGH provider.Executor (the P1.M2.T4.S1 contract). The stub is consumed by the e2e suite (P1.M6.T3.S2) and the §18.1 invariant tests (P1.M6.T3.S3). PRD §20.1 layer 3; decisions.md §8. stdlib + internal/provider only; no testify; no real LLM. Scope: deliver the stub asset + self-tests ONLY — do NOT create generate.go/CommitStaged (T1.S1) or the e2e/invariant suites.

## Context

## Goal

**Feature Goal**: A reusable, self-tested **fake coding-agent CLI** (a tiny `package main` binary compiled from `internal/generate/testdata/stubagent`) plus a `package generate` test harness (`internal/generate/stubprovider_test.go`) that exposes `BuildStubBinary(t)` and `NewStubManifest(t, StubConfig)`, so the generate integration suite (P1.M6.T3.S2) and the §18.1 invariant/property tests (P1.M6.T3.S3) can drive `generate.CommitStaged` end-to-end with deterministic, scriptable agent behavior — emitting a canned valid message, empty/garbage (parse-correction retry then success/rescue), a subject duplicating a known recent commit (outer dup loop), or hanging until killed (timeout). Replaces the real agent entirely (PRD §20.1 layer 3; decisions.md §8).

**Deliverable**: Two new files only — (1) `internal/generate/testdata/stubagent/main.go` (`package main`, stdlib-only): reads stdin (optionally logging it), reads a JSON response "script" + a counter "state" file from env, and per-invocation emits the scripted response (message / empty / garbage / hang / non-zero exit); (2) `internal/generate/stubprovider_test.go` (`package generate`): `BuildStubBinary`, `NewStubManifest`, `StubConfig`, `StubResponse`, and a `TestStubProvider_*` self-test suite that asserts the stub emits each configured behavior THROUGH the real `provider.Executor` (valid, stateful empty→valid, hang→`*TimeoutError`, Fail→`*AgentError`, stdin capture, dup-then-unique ordering).

**Success Definition**: `gofmt`, `go build ./internal/generate/`, `go build ./internal/generate/testdata/stubagent`, `go vet`, and `go test ./internal/generate/ -run StubProvider` all pass; the self-tests prove every contract behavior mode; and a downstream test in `package generate` can produce a fully-wired `provider.Manifest` with one `NewStubManifest(t, cfg)` call.

## Why

- **Unblocks the core-IP proof**: the snapshot-based atomic-commit pipeline (PRD §13/§18.1) and its two-nested-loop `CommitStaged` (reference_impl.md §2) cannot be proven end-to-end against a real LLM in CI. A deterministic fake agent is the only way to exercise every path (success, dup-retry, parse-retry, timeout, rescue) reproducibly (decisions.md §8).
- **Real LLM is opt-in/manual**: per PRD §20.1 layer 4, real-agent runs live behind `//go:build integration_real` + `STAGEHAND_RUN_REAL=1` and run only before releases. The stub is what CI runs.
- **Isolation**: the stub stands as its OWN unit (plan_overview §8 — "the genuinely-separate test harness/infra: stub provider, temp-repo util, invariant proofs"), so M6.T3.S2/S3 compose it instead of reinventing a fake agent each.

## Implementation Blueprint

### testdata/stubagent/main.go (package main, stdlib-only)
```go
package main
import (
	"encoding/json"
	"fmt"
	"io"
	"os"
	"strconv"
	"strings"
)
type stubResponse struct {
	Emit string `json:"emit"` 
	Hang bool   `json:"hang"` 
	Fail int    `json:"fail"` 
}
func main() {
	in, _ := io.ReadAll(os.Stdin)
	if p := os.Getenv("STAGEHAND_STUB_STDIN"); p != "" {
		_ = os.WriteFile(p, in, 0o644)
	}
	script := []stubResponse{{Emit: "feat: stubbed commit message"}}
	if s := os.Getenv("STAGEHAND_STUB_SCRIPT"); s != "" {
		if err := json.Unmarshal([]byte(s), &script); err != nil {
			fmt.Fprintf(os.Stderr, "stubagent: invalid STAGEHAND_STUB_SCRIPT: %v\n", err)
			os.Exit(1)
		}
	}
	idx := 0
	if sp := os.Getenv("STAGEHAND_STUB_STATE"); sp != "" {
		if b, err := os.ReadFile(sp); err == nil {
			if n, err := strconv.Atoi(strings.TrimSpace(string(b))); err == nil {
				idx = n
			}
		}
		_ = os.WriteFile(sp, []byte(strconv.Itoa(idx+1)), 0o644) 
	}
	if idx >= len(script) {
		idx = len(script) - 1
	}
	r := script[idx]
	if r.Hang {
		select {} 
	}
	if r.Fail > 0 {
		fmt.Fprintf(os.Stderr, "stubagent: simulated agent failure (exit %d)\n", r.Fail)
		os.Exit(r.Fail)
	}
	fmt.Fprint(os.Stdout, r.Emit)
}

```

### stubprovider_test.go (package generate, white-box)
```go
package generate
import (
	"context"
	"encoding/json"
	"errors"
	"fmt"
	"os"
	"os/exec"
	"path/filepath"
	"strings"
	"sync"
	"testing"
	"time"
	"github.com/dustin/stagehand/internal/provider"
)
type StubResponse struct {
	Emit string `json:"emit"`
	Hang bool   `json:"hang"`
	Fail int    `json:"fail"`
}
type StubConfig struct {
	Script    []StubResponse
	StateFile string
	StdinLog  string
}
var (
	stubBinOnce sync.Once
	stubBinPath string
	stubBinErr  error
)
func BuildStubBinary(t testing.TB) string {
	t.Helper()
	stubBinOnce.Do(func() {
		dir, err := os.MkdirTemp("", "stagehand-stub-*")
		if err != nil { stubBinErr = err; return }
		stubBinPath = filepath.Join(dir, "stubagent")
		cmd := exec.Command("go", "build", "-o", stubBinPath, "./testdata/stubagent")
		if out, err := cmd.CombinedOutput(); err != nil {
			stubBinErr = fmt.Errorf("go build stubagent: %w\n%s", err, out)
		}
	})
	if stubBinErr != nil {
		t.Fatalf("build stub binary: %v", stubBinErr)
	}
	return stubBinPath
}
func NewStubManifest(t testing.TB, cfg StubConfig) provider.Manifest {
	t.Helper()
	if len(cfg.Script) == 0 {
		t.Fatal("NewStubManifest: cfg.Script must be non-empty")
	}
	b, err := json.Marshal(cfg.Script)
	if err != nil {
		t.Fatalf("marshal stub script: %v", err)
	}
	env := map[string]string{"STAGEHAND_STUB_SCRIPT": string(b)}
	if cfg.StateFile != "" {
		env["STAGEHAND_STUB_STATE"] = cfg.StateFile
	}
	if cfg.StdinLog != "" {
		env["STAGEHAND_STUB_STDIN"] = cfg.StdinLog
	}
	return provider.Manifest{
		Name:           "stub",
		Command:        BuildStubBinary(t),
		PromptDelivery: provider.DeliveryStdin,
		Env:            env,
	}
}

```

## Anti-Patterns to Avoid
- ❌ Don't configure the stub via os.Setenv (leaks across subtests) — use manifest Env (Gotcha #2).
- ❌ Don't keep per-call state in-process memory (fresh process per call) — use a counter file (Gotcha #3).
- ❌ Don't wrap the counter on overflow — CLAMP to the last entry for determinism.
- ❌ Don't busy-loop on Hang — `select {}` so the group kill fells it; assert *TimeoutError.
- ❌ Don't let the testdata struct drift from StubResponse JSON tags — silent wrong behavior.
- ❌ Don't create generate.go / CommitStaged / the e2e suite / invariant tests in this task (out of scope).
- ❌ Don't add testify or external deps (codebase is stdlib-only).
- ❌ Don't skip reading stdin before acting (Executor's pipe must drain — Gotcha).
